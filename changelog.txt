Date: 27/03/2026
----------------
Change Type: Modification
Description: Updated NCR alert thresholds with defect-mode-specific rules
Details:
  - Updated Summary.py (src/Pages/Summary.py)
    - Changed checkDefectRatios() to apply per-mode alert thresholds instead of a flat 5% rule:
      - ROUGHCUT: triggers alert when defect ratio > 20%
      - LOUT: triggers alert when total defects > 1 piece
      - All other defect modes: retain existing > 5% ratio rule
    - Highlighted out-of-spec rows are still coloured light red (#ffcccc)
    - Updated warning popup message to list all three threshold rules clearly
    - Updated email prompt text to reference "alert limits" instead of "5% threshold"
    - Updated email details key from "Defect Modes Exceeding 5%" to "Defect Modes Exceeding Alert Limits"
    - Updated email subject/error_type to "NCR Alert - Defect Alert Limits Exceeded"

Date: 02/03/2026
----------------
Change Type: Addition
Description: Added Evaluation Lot Mode toggle with credential verification
Details:
  - Added Evaluation Lot Mode toggle switch in MainPage.py (src/Pages/MainPage.py)
    - Toggle switch labeled "Eval Mode" placed beside Input Quantity field
    - Uses standard font size (font["L"]) for better visibility
    - Requires credential verification (same as Settings page) before enabling
  - Evaluation Mode functionality:
    - When enabled, all defect types become manually inputtable (similar to DROPCHIP/SAMPLE)
    - Converts defect labels to clickable buttons for manual input
    - When disabled, returns to normal automatic defect detection mode
  - Image capture in Evaluation Mode:
    - Captures and saves images only (skips automatic defect detection)
    - Displays captured image to user
    - Initializes Excel file structure for manual defect entry
    - Shows message prompting user to manually input defect quantities
  - Credential verification:
    - Created checkEvalCredentials() method for login verification
    - Uses same username/password credentials as Settings page access
    - Toggle remains unchecked if credentials fail
  - UI improvements:
    - Toggle positioned beside Input Quantity field for better accessibility
    - Toggle state persists across resets
